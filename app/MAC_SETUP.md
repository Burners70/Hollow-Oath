# Hollow Oath — iOS wrapper setup (run on the Mac)

Everything in `app/` was authored so that turning the web game into the iOS
app is a short checklist on a Mac. The web game at the repo root stays the
source of truth; `app/www` is a generated copy (`sync.sh`), and the Xcode
project is generated by Capacitor and then configured by `configure-ios.sh`.

Roadmap coverage: **E1/E2** (scaffold + shell config), **E4** (iCloud
key-value save sync), **E5** (privacy manifest), **E6** (icon + launch
screen), **G1–G4** (Game Center) are code-complete. **E8** (device matrix)
and **F3** (haptics restraint pass) are the on-device steps at the end of
this checklist.

## 1. Prereqs

- Xcode 15+ (App Store), opened once so the license is accepted
- Node 18+ (`brew install node`)
- CocoaPods (`brew install cocoapods`)
- An Apple Developer Program membership (for device runs, TestFlight, submission)

## 2. One-shot bootstrap

```sh
cd app
./setup-mac.sh
```

This runs `npm install`, copies the web game into `www/`, generates the
Xcode project (`npx cap add ios`) if it doesn't exist, then applies the
native shell config (`configure-ios.sh`):

- landscape-only, status bar hidden, `UIRequiresFullScreen` (E2)
- iOS 16.0 deployment target (owner decision — perf floor at A11+)
- `App.entitlements` — Game Center + iCloud key-value storage
- `PrivacyInfo.xcprivacy` — no tracking, no data collected (E5)
- single-size 1024 AppIcon + black wordmark launch screen (E6)

Re-run `./setup-mac.sh` any time; it is idempotent. After changing the web
game, refresh the wrapper with `npm run sync`.

## 3. Signing (Xcode, one time)

`npx cap open ios` → select the **App** target → *Signing & Capabilities*:

1. Pick your team; leave "Automatically manage signing" on.
2. Bundle ID must read `com.burners70.hollowoath` (set by Capacitor —
   **permanent once shipped, do not change**).

## 4. Capabilities (App ID)

The entitlements file is already wired in. With automatic signing, Xcode
registers the App ID and enables what the entitlements ask for — verify in
*Signing & Capabilities* that you see **no red errors** for:

- **iCloud → Key-value storage** (E4 save sync)
- **Game Center** (G)

If Xcode complains, press "+ Capability" and add *iCloud* (check only
"Key-value storage") and *Game Center* once — that reconciles the App ID —
then confirm `App/App.entitlements` still contains both keys.

## 5. App Store Connect — Game Center records (G2/G3)

Create the app record (name **Hollow Oath**, bundle
`com.burners70.hollowoath`), then under *Services → Game Center* create
**exactly these IDs** (they must match `GC_BOARD_*`/`GC_ACH` in
`index.html`):

Leaderboards (classic, integer score, best-score-wins):

| ID | Name |
|---|---|
| `hollowoath.score.alltime` | ROTATION RECORD (all-time) |
| `hollowoath.score.daily` | DAILY FLIGHT |

FIELD MEDIC (easy mode) runs never post to either board — gated in
`saveHi()`/`recordDaily()` (H3).

Achievements (all 100-pt single-step, hidden until earned where noted).
The 1024×1024 images and the earned/pre-earned description copy to paste
into each record are in `assets/gamecenter/achievements/` and
[../GAMECENTER_ACHIEVEMENTS.md](../GAMECENTER_ACHIEVEMENTS.md):

| `hollowoath.oath_keeper` | OATH KEEPER | answered ending, zero shots fired |
| `hollowoath.hollow_keeper` | HOLLOW KEEPER (hidden) | answered; fired only at secrets, never in combat |
| `hollowoath.the_one_who_answered` | THE ONE WHO ANSWERED | answered ending, any fire |

## 6. First device run — what to verify

- Game boots full-screen landscape, black behind the webview, no status bar.
- **Haptics** (F1/F2): boarding lub-dub vs. the saboteur's single dull thud;
  the extraction double-tick; hard landing thud. (`window.Capacitor` present
  → the facade bridges automatically.)
- **A2HS banner and fullscreen prompts are absent** (E3's `NATIVE` switch).
- **Auto-pause**: background the app mid-flight → reopen → pause screen, run
  intact (A4 + the native `appStateChange` hook).
- **iCloud sync** (E4): play a sector on device A, install on device B (same
  Apple ID) → hi-score/codex/logs appear; a mid-run save offers RESUME only
  if device B has no local run.
- **Game Center**: sign-in sheet appears once; finish a run → score on the
  all-time board; achievements unlock per the table above.
- Silent-switch behaviour: game audio should respect the ringer switch
  (WKWebView default) — confirm and note it in the E8 checklist.
- **TILT is pulled entirely for this release (owner decision, July 2026,
  found on-device).** `DeviceOrientationEvent.requestPermission()` has no
  system prompt to grant against inside a bare Capacitor WKWebView (no
  per-site permission store the way Safari has) — it silently denies every
  time, `NSMotionUsageDescription` or not. Rather than ship a half-working
  toggle, the Settings row is removed outright (`SETTINGS_ROWS` 10 → 9 in
  `js/world.js`; `js/render.js`/`js/update.js` reindexed to match) and the
  plist key isn't declared (`app/configure-ios.sh`) — an unused permission
  is its own App Review flag. Touch and gamepad steering are unaffected. The
  underlying gyro plumbing (`tilt`/`gyro`/`enableGyro`/`toggleTilt` in
  `js/input.js`) is left in place, just unreachable from the UI — re-add the
  Settings row and the plist key together once a real CoreMotion-backed
  native plugin exists (same pattern as haptics/Game Center/iCloud).

## 8. F3 — haptics restraint pass (on device, with E8)

Play a full sector. If haptics fire more than ~once per 5 s of normal
flight, cut the least meaningful call sites. First candidates (per the
roadmap): the intro-screen heartbeat, the boarding lub-dub for ordinary
Scions. Haptics carry meaning here only if they stay rare.

## Troubleshooting

- **Plugins missing in `npx cap ls`** — run `npm install` in `app/` first;
  the two local plugins are `file:` dependencies and must be present in
  `node_modules` before `cap sync`.
- **`Capacitor.Plugins.ICloudKV`/`GameConnect` undefined at runtime** — the
  plugin class list is generated at `cap sync`; re-run `npm run sync` and
  rebuild. Both facades no-op silently when the bridge is absent, so the
  game itself will still run.
- **Pod install fails on deployment target** — `configure-ios.sh` pins the
  Podfile and project to iOS 16.0; re-run it after regenerating `ios/`.
- **iCloud writes don't sync** — NSUbiquitousKeyValueStore needs the
  key-value entitlement AND an iCloud-signed-in device; the simulator is
  unreliable for this, test on hardware.
